This directory of the Web Vulnerabilities Checklist documents cookie-related vulnerabilities: session fixation, cookie injection, parameter pollution, cookie bomb DoS, mass assignment via cookies, SQLi in cookies, XSS via cookies, privilege escalation, session puzzling, and Python pickle injection. The full reference is in [cookie.md](./cookie.md).

## Synonyms

Cookie attack, session fixation, cookie injection, cookie tampering, session hijacking, session puzzling, cookie bomb, denial of service via cookies, cookie parameter pollution, SQL injection in cookies, XSS via cookies, privilege escalation via cookies, mass assignment via cookies, weak session management, missing HttpOnly, missing Secure, missing SameSite.

## Attack patterns covered

Sensitive data in cookies (Base64, hex, MD5, JWT), buffer overflow on oversized cookies, arbitrary cookie injection via `Set-Cookie` reflection, mass assignment via cookies (`is_admin=true`, `role=admin`), cookie bomb DoS, SQL injection in cookie values, cookie parameter pollution (same name twice), authentication bypass via cookie tampering, XSS via reflected cookie value, missing session invalidation on logout, missing session regeneration on privilege change, weak session timeout, missing `HttpOnly` / `Secure` / `SameSite` attributes, horizontal and vertical privilege escalation, session puzzling, Python pickle injection in cookies.

## Programs where cookie bugs pay

HackerOne, Bugcrowd, Intigriti, YesWeHack, Synack, programs with web sessions, single sign-on (SSO), OAuth integrations, multi-tenant SaaS.
