This directory of the Web Vulnerabilities Checklist documents password reset attack techniques: token reuse, host header injection, HTML injection, referer leak, corporate invite abuse, CRLF injection, token manipulation, SQL injection, response manipulation, oversized tokens, cross-domain token leak, race conditions and TLD squatting. ## Synonyms

Password reset attack, password reset poisoning, host header injection, reset token reuse, reset link leak, password reset bypass, account recovery attack, password recovery vulnerability, forgot password bug.

## Attack patterns covered

Token reuse (same token usable multiple times, or after password change), host header injection (poison the reset link to attacker domain), HTML injection in reset email, referer leakage of reset token to third-party domains, corporate-email invite abuse, CRLF injection in reset request, token manipulation tricks (truncation, length extension, encoding swap), SQL injection in reset endpoints, response status manipulation, oversized token DoS, cross-domain token leak via redirects, race conditions (parallel reset requests, parallel use of same token), TLD squatting on reset email links, predictable token (timestamp / counter / weak RNG), missing token expiry, IDOR on reset endpoint, password-reset auto-login session-fixation pattern.
