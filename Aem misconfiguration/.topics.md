# AEM (Adobe Experience Manager) Misconfiguration â€” CMS Vulnerability Reference

This directory of the Web Vulnerabilities Checklist documents vulnerabilities and misconfigurations in Adobe Experience Manager (AEM): Sling dispatcher bypass, servlet abuse, SSRF, Groovy console RCE, JCR secret extraction, and historical AEM CVEs. The full reference is in [aem.md](./aem.md).

## Synonyms

AEM hacking, Adobe Experience Manager hacking, AEM misconfiguration, AEM dispatcher bypass, AEM servlet abuse, AEM SSRF, AEM RCE, Groovy console exploitation, JCR query abuse, QueryBuilder abuse, Sling servlet bypass.

## Attack patterns covered

Dispatcher selector / extension tricks (`.css/a.html`, `.json/a.css`, `.json;%0a.css`, `.json#a.css`), `WCMDebugFilter` XSS (CVE-2016-7882), `cqdamcomponents` XSS, `IconUriServlet` SSRF (CVE-2016-0957 family), Opensocial proxy SSRF (`/libs/opensocial/proxy`), Salesforce OAuth SSRF, dispatcher invalidate SSRF, Groovy console RCE (`/system/console/groovyconsole`), JCR secret extraction via QueryBuilder (`/bin/querybuilder.json`), AEM hacker tool, useful AEM fingerprinting endpoints (`/system/console/bundles`, `/etc/clientcontext/default/contentloader.json`, `/libs/granite/security/currentuser.json`).

## Programs where AEM bugs pay

HackerOne, Bugcrowd, Intigriti, YesWeHack, Adobe Bug Bounty, enterprise programs running AEM (banks, governments, telcos, retailers).
