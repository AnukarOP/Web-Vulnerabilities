This directory of the Web Vulnerabilities Checklist documents Mass Assignment vulnerabilities: injecting `is_admin`, `role`, `user_priv`, `isadmin`, `ADMIN`, `administrator`, and organization-takeover fields during registration, password reset, profile update and email change requests. The full reference is in [Mass.md](./Mass.md).

## Synonyms

Mass assignment, parameter binding vulnerability, autobinding flaw, over-posting, Rails strong parameters bypass, Django `__all__` form bypass, Spring `@ModelAttribute` over-posting, API3 broken object property level authorization, privilege escalation via mass assignment.

## Attack patterns covered

`admin: true` injection during registration, `ADMIN: true`, `isadmin: true`, `ISADMIN: true`, `Admin: true` case variants, `role: admin` / `role: ADMIN` / `role: administrator` / `user_priv: admin` / `user_priv: administrator`, `admin: 1` numeric variant, unauthorized organization access (`organization`, `org`, `org_id`, `tenant`, `tenant_id`, `company_id`, `team_id`, `workspace_id`, `project_id`), email-change mass assignment, password-reset mass assignment, login mass assignment, profile-update mass assignment.
