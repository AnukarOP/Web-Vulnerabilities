# Authentication â€” Login Security Reference

This directory of the Web Vulnerabilities Checklist documents authentication-related vulnerabilities: authentication bypass, captcha bypass, weak password policy, missing rate limit, user enumeration, missing password re-confirmation, and HTTPS-not-enforced login. The full reference is in [authentication.md](./authentication.md).

## Synonyms

Authentication bypass, login bypass, captcha bypass, weak password policy, user enumeration, username enumeration, email enumeration, missing rate limit, brute force, credential stuffing, insecure login, HTTP login, password policy bypass.

## Attack patterns covered

Direct access to authenticated URLs without session, captcha bypass via `X-Forwarded-For`, missing password re-confirmation before sensitive actions (email change, password change, account deletion, 2FA management), no rate limit on login, no captcha on login, username / email enumeration via response differential, weak password policy (numeric-only, lowercase-only, dictionary passwords, short passwords), credentials sent over HTTP, login form CSRF.

## Programs where auth bugs pay

HackerOne, Bugcrowd, Intigriti, YesWeHack, Synack, programs with web login portals, mobile-app back-ends, SaaS dashboards.
