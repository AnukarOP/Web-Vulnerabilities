This directory of the Web Vulnerabilities Checklist documents techniques to bypass rate limits, brute-force protections, throttling and account-lockout mechanisms: null / control character injection, host header injection, cookie rotation, IP spoofing headers and double `X-Forwarded-For`. The full reference is in [bypass rate limit.md](./bypass%20rate%20limit.md).

## Synonyms

Rate limit bypass, throttling bypass, brute force protection bypass, account lockout bypass, captcha bypass, anti-bot bypass, IP-based rate limit bypass, request-rate evasion, denial of service.

## Attack patterns covered

Null / control character injection (`%00`, `%0d`, `%0a`, `%09`, `%20`), host header injection, cookie / session rotation between attempts, originating-IP spoofing via headers (`X-Forwarded-For`, `X-Originating-IP`, `X-Remote-IP`, `X-Client-IP`, `X-Real-IP`, `X-Forwarded`, `Forwarded`, `Client-IP`, `True-Client-IP`, `CF-Connecting-IP`), double `X-Forwarded-For` with different IPs, IPv4-to-IPv6 mapping (`::ffff:127.0.0.1`), HTTP/2 â†’ HTTP/1.1 confusion at the edge, requesting the rate-limited endpoint at slightly different paths (`/api/login`, `/api/login/`, `/api/Login`, `/api/login%20`).
