# Remote Code Execution (RCE) â€” Exploitation Path Reference

This directory of the Web Vulnerabilities Checklist documents Remote Code Execution (RCE) pivot paths from other vulnerability classes: RCE via dependency confusion, file upload, SQL injection, LFI, SSRF, XXE, command injection, insecure deserialization and Server-Side Template Injection (SSTI). The full reference is in [RCE.md](./RCE.md).

## Synonyms

Remote code execution, RCE, arbitrary code execution, command injection, OS command injection, code injection, server-side code execution, unauthenticated RCE, pre-auth RCE, post-auth RCE, full system compromise, shell on the server.

## Attack patterns covered

RCE via dependency confusion (npm / PyPI / RubyGems internal-package hijack), RCE via file upload (webshell, `.htaccess`, `web.config`, SVG, ImageTragick), RCE via SQL injection (`xp_cmdshell`, `LOAD_FILE` / `INTO OUTFILE`, PostgreSQL `COPY ... TO PROGRAM`), RCE via LFI to log poisoning / `/proc/self/environ`, RCE via SSRF to Redis / Memcached / Docker / Kubernetes / cloud metadata, RCE via XXE to OOB and PHP `expect://` wrapper, RCE via command injection (back-tick, `;`, `&&`, `||`, `|`, `$()`, `${IFS}`), RCE via insecure deserialization (Java `ysoserial`, Python `pickle`, PHP `unserialize` with `phpggc`, .NET `BinaryFormatter`, Ruby `Marshal.load`), RCE via SSTI (Jinja2, Twig, Freemarker, Velocity, ERB, Smarty, Handlebars, Liquid).
