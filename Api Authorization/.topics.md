# API Authorization â€” BOLA and IDOR Pattern Reference

This directory of the Web Vulnerabilities Checklist documents Broken Object Level Authorization (BOLA) and Insecure Direct Object Reference (IDOR) patterns in REST and GraphQL APIs: predictable IDs, ID combinations, integer/email IDs, group IDs, nested-object IDs, CRLF in IDs, array smuggling, parameter pollution. The full reference is in [Authorization.md](./Authorization.md).

## Synonyms

API authorization, BOLA, Broken Object Level Authorization, IDOR, Insecure Direct Object Reference, API1, API3, object-level access, function-level access, horizontal privilege escalation, vertical privilege escalation, multi-tenancy bypass, tenant isolation bypass.

## Attack patterns covered

Predictable integer IDs, UUID prediction, integer vs string ID swap, email as ID, group ID swap, nested object ID injection (`{"Account": {"Account": 3333}}`), multiple-ID submission, predictable session token, CRLF in ID (`%0d%0a`), LF in ID (`%0a`), CR in ID (`%0d`), null byte in ID (`%00`), array smuggling (`[victim, attacker]`), semicolon-separated IDs, JSON array of IDs in the same key, HTTP parameter pollution.

## Programs where BOLA / API authorization bugs pay

HackerOne, Bugcrowd, Intigriti, YesWeHack, Synack, programs running REST APIs, GraphQL APIs, mobile-app back-ends, SaaS multi-tenant platforms.
